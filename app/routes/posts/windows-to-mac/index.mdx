## はじめに

Windowsを長年使ってきたユーザがMacに乗り換えると、**キーボード配列の違い**や**ウィンドウ管理の仕組み**など、さまざまな点で戸惑いが生じます。しかし、適切な設定とアプリを導入すれば、Windowsに近い操作感を実現しつつ、Macならではの快適さも享受できます。

本記事では、筆者がWindows → Mac に移行した際に行った設定変更やインストールしたアプリを紹介します。

<Note>

本記事の内容は **macOS Sequoia（15.x）** 以降を前提としています。バージョンによって設定項目の場所が若干異なる場合があります。

</Note>

## まず最初にやるべきシステム設定

### トラックパッドの設定

Macのトラックパッドは非常に優秀ですが、Windowsのタッチパッドとスクロール方向が逆（ナチュラルスクロール）になっています。

**システム設定 → トラックパッド → スクロールとズーム**

- 「ナチュラルなスクロール」を **オフ** にするとWindowsと同じスクロール方向になります

<Note>

ナチュラルスクロールに慣れるのもおすすめです。スマホと同じ感覚で操作できるため、慣れると自然に感じます。

</Note>

また、**タップでクリック**も有効にしておくと便利です。

**システム設定 → トラックパッド → ポイントとクリック**

- 「タップでクリック」を **オン**

### Dock の設定

Dockは初期状態では大きくて場所を取ります。以下の設定がおすすめです。

**システム設定 → デスクトップとDock**

| 設定項目 | おすすめ値 |
|---|---|
| サイズ | 小さめに調整 |
| 拡大 | オフ、もしくは小さめ |
| 画面上の位置 | 左（ディスプレイの縦幅を広く使える） |
| Dockを自動的に表示/非表示 | オン |
| 最近使ったアプリをDockに表示 | オフ |

### マウスの速度

デフォルトではカーソル速度が遅いと感じることが多いです。

**システム設定 → マウス（またはトラックパッド）→ 軌跡の速さ**

- 最大に近い値に設定するのがおすすめ

## キーボード・入力まわりの設定

WindowsユーザがMacで最も戸惑うのがキーボードです。特に **Command ⌘** キーと **Control ⌃** キーの役割の違いは大きなハードルです。

### 修飾キーの変更

Windowsで `Ctrl + C` だったコピーは、Macでは `⌘ + C` になります。物理的なキー位置が異なるため、以下の変更がおすすめです。

**システム設定 → キーボード → キーボードショートカット → 修飾キー**

| 変更前 | 変更後 |
|---|---|
| Caps Lock ⇪ | Control ⌃ |
| Control ⌃ | Command ⌘（任意） |

<Note>

Caps Lockキーは使用頻度が低いため、Controlに置き換えるとターミナル操作が非常に快適になります。これはMacユーザの中でも非常に一般的なカスタマイズです。

</Note>

### 入力ソースの設定

**システム設定 → キーボード → 入力ソース → 編集**

- **ライブ変換** を **オフ** にする（Windowsのように確定前にスペースキーで変換する方式になる）
- **Windows風のキー操作** を **オン** にする（利用可能な場合）

### 自動補完・自動修正をオフにする

Macでは英語入力時にスペルの自動修正や単語の自動補完が有効になっており、意図しない文字に勝手に置き換わることがあります。Windowsにはない挙動のため、戸惑うユーザが非常に多いです。

**システム設定 → キーボード → 入力ソース → 編集**（または **すべての入力ソース** セクション）

以下の項目をすべて **オフ** にします。

| 設定項目 | 説明 |
|---|---|
| スペルを自動的に修正 | 英語入力時にスペルミスが勝手に修正される |
| 文頭を自動的に大文字にする | 文の先頭が自動で大文字になる |
| スペースバーを2回押してピリオドを入力 | スペース2回で `.` が入力される |
| スマート引用符とスマートダッシュを使用 | `"` が `""`（カーリークォート）に自動変換される |

<Note>

特に **スペルの自動修正** と **スマート引用符** は、プログラミングやコマンド入力時に深刻な問題を引き起こします。コード中の引用符が勝手にカーリークォートに変換されてエラーになるケースは非常にありがちです。開発者は必ずオフにしましょう。

</Note>

ターミナルからまとめて無効化することもできます。

```bash
# スペルの自動修正をオフ
defaults write -g NSAutomaticSpellingCorrectionEnabled -bool false

# 文頭の自動大文字化をオフ
defaults write -g NSAutomaticCapitalizationEnabled -bool false

# スマート引用符をオフ
defaults write -g NSAutomaticQuoteSubstitutionEnabled -bool false

# スマートダッシュをオフ
defaults write -g NSAutomaticDashSubstitutionEnabled -bool false

# スペースバー2回でピリオドをオフ
defaults write -g NSAutomaticPeriodSubstitutionEnabled -bool false
```

### キーリピートの高速化

デフォルトではキーリピートが遅いため、ターミナルやテキスト編集で不便です。

**システム設定 → キーボード**

- 「キーのリピート速度」を **最速**
- 「リピート入力認識までの時間」を **最短**

さらに高速にしたい場合は、ターミナルで以下を実行します。

```bash
# キーリピート速度（デフォルト: 2、小さいほど速い）
defaults write -g KeyRepeat -int 1

# リピート開始までの時間（デフォルト: 15、小さいほど速い）
defaults write -g InitialKeyRepeat -int 10
```

## ファンクションキーの使い方

Windowsではファンクションキー（F1〜F12）をそのまま押せば機能が発動しますが、Macでは**デフォルトでメディアキー（輝度・音量など）が割り当てられている**ため、`fn` キーを同時に押す必要があります。

### Windows との主な違い

Windowsユーザが特に戸惑いやすいファンクションキーの違いをまとめます。

| 操作 | Windows | Mac |
|---|---|---|
| ファイル名の変更 | `F2` | `Enter`（Finderで選択中にEnter） |
| 検索 | `F3` | `⌘ + F` |
| アドレスバーに移動 | `F4` / `Alt + D` | `⌘ + L` |
| ページの更新 | `F5` | `⌘ + R` |
| フルスクリーン切替 | `F11` | `⌃ + ⌘ + F` |
| 開発者ツール | `F12` | `⌘ + ⌥ + I` |

<Note>

Macの Finder では **`Enter` キーがファイル名の変更**（Windowsの `F2` に相当）に割り当てられています。ファイルを開くには `⌘ + O` または `⌘ + ↓` を使います。これはWindowsユーザが最も戸惑うポイントの一つです。

</Note>

### Touch Bar 搭載モデルの場合

一部の MacBook Pro（2016〜2020年モデル）では、物理的なファンクションキーの代わりに **Touch Bar** が搭載されています。Touch Bar では `fn` キーを押している間だけファンクションキーが表示されます。

特定のアプリで常にファンクションキーを表示したい場合は以下で設定できます。

**システム設定 → キーボード → キーボードショートカット → ファンクションキー**

アプリごとに「F1、F2 などのキーを標準のファンクションキーとして使用」するアプリを追加（`+` ボタン）できます。ターミナルやIDEなど、Fキーを頻繁に使うアプリを登録しておくと便利です。

## Finder の設定

FinderはWindowsのエクスプローラーに相当しますが、デフォルトでは使いにくい部分があります。

### パスバーを表示

**Finder → 表示 → パスバーを表示**（または `⌥⌘P`）

現在のディレクトリのフルパスが下部に表示され、どこにいるか把握しやすくなります。

### ステータスバーを表示

**Finder → 表示 → ステータスバーを表示**（または `⌘/`）

ファイル数やディスク容量が確認できます。

### 拡張子を常に表示

**Finder → 設定 → 詳細 → すべてのファイル名拡張子を表示**

Windowsと同様に拡張子が常に表示されるようになります。

### 隠しファイルを表示

Finderで `⌘⇧.`（Command + Shift + ドット）を押すと、隠しファイルを表示できます。

常に表示したい場合は、ターミナルで以下を実行します。

```bash
defaults write com.apple.finder AppleShowAllFiles -bool true
killall Finder
```

### デフォルトの表示をリスト表示に変更

**Finder → 表示 → リスト**（または `⌘2`）

Windowsの詳細表示に近い レイアウトになります。

## インストールすべきおすすめアプリ

### パッケージマネージャ：Homebrew

Macで最初にインストールすべきは **Homebrew** です。Windowsにおける winget や Chocolatey に相当します。

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

以降のアプリの多くは `brew install` でインストール可能です。

<BlockLink href="https://brew.sh/ja/">
  Homebrew — macOS用パッケージマネージャ
</BlockLink>

### ウィンドウ管理：Rectangle

Windowsでは `Win + 矢印キー` でウィンドウを左右に分割できますが、Macにはデフォルトでこの機能がありません（macOS Sequoia以降は標準で対応）。**Rectangle** を導入すると、Windowsライクなウィンドウスナップが使えます。

```bash
brew install --cask rectangle
```

<BlockLink href="https://rectangleapp.com/">
  Rectangle — Move & resize windows in macOS
</BlockLink>

<Note>

macOS Sequoia（15.0）以降では、標準のウィンドウタイリング機能がかなり改善されました。まずは標準機能を試し、不足があればRectangleを導入するとよいでしょう。

</Note>


### キーボードカスタマイズ：Karabiner-Elements

キーボードのリマッピングを細かく制御できるツールです。たとえば「英数/かな」キーの挙動変更や、Windows配列キーボードをMac風に使う設定が可能です。

```bash
brew install --cask karabiner-elements
```

#### Simple Modifications（単純なキー置換）

Karabiner-Elements の設定画面から **Simple Modifications** タブを開き、以下のようなキー置換を設定します。

| 変更前（From key） | 変更後（To key） | 用途 |
|---|---|---|
| `caps_lock` | `left_control` | Caps LockをControlとして使う |
| `right_command` | `right_option` | Windows用キーボード向け調整 |


<BlockLink href="https://karabiner-elements.pqrs.org/">
  Karabiner-Elements — A powerful keyboard customizer for macOS
</BlockLink>

<BlockLink href="https://ke-complex-modifications.pqrs.org/">
  Karabiner-Elements Complex Modifications Rules
</BlockLink>

### 圧縮・展開：The Unarchiver

Windowsで作成されたZIPファイルの文字化けを防ぐためにも、The Unarchiverの導入がおすすめです。

```bash
brew install --cask the-unarchiver
```

### ブラウザ：Google Chrome

Safari も優秀ですが、Windowsから移行する場合はブックマークや拡張機能の引き継ぎのために **Chrome** を入れておくと安心です。

```bash
brew install --cask google-chrome
```

### システムモニター：RunCat

**RunCat** は、メニューバーに走る猫のアニメーションでCPU使用率を可視化してくれるアプリです。CPU負荷が高いほど猫が速く走り、一目でシステムの状態がわかります。

Windowsのタスクマネージャーのように常時CPU・メモリ使用率を確認する習慣があるユーザにとって、メニューバーから手軽にリソース状況を把握できるのは便利です。

App Store からインストールできます。

<BlockLink href="https://apps.apple.com/jp/app/runcat/id1429033973">
  RunCat — App Store
</BlockLink>

主な機能：
- **CPU使用率** をアニメーション速度で直感的に表示
- メニューバーをクリックすると **CPU / メモリ / ディスク / ネットワーク** の詳細情報を確認可能
- 猫以外にも **犬・鳥・うさぎ** などアニメーションの種類を変更可能
- バッテリー残量やストレージ使用量の表示にも対応

<Note>

RunCat は無料で基本機能が使えます。追加のアニメーションや詳細なモニタリング機能は App 内課金で利用可能です。

</Note>

## 開発者向けツール

### ターミナル環境

Mac標準の **Terminal.app** も使えますが、以下のターミナルアプリが人気です。

```bash
# iTerm2（定番）
brew install --cask iterm2

# Warp（モダンで高速）
brew install --cask warp

# Ghostty（軽量で高速）
brew install --cask ghostty
```

### シェル：zsh + Oh My Zsh

macOSではzshがデフォルトシェルです。**Oh My Zsh** を導入すると、プラグインやテーマで快適に拡張できます。

```bash
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
```

おすすめプラグイン：
- **zsh-autosuggestions**（コマンド補完）
- **zsh-syntax-highlighting**（構文ハイライト）

### Git

```bash
brew install git
```

### エディタ：Visual Studio Code

```bash
brew install --cask visual-studio-code
```

Windowsで使っていた設定は **Settings Sync** 機能で引き継ぎ可能です。

### Docker

```bash
brew install --cask docker
```

## その他の便利な設定

### 3本指ドラッグの有効化

ウィンドウの移動やテキスト選択が楽になります。

**システム設定 → アクセシビリティ → ポインタコントロール → トラックパッドオプション → ドラッグ方法 → 3本指のドラッグ**

### ホットコーナーの無効化

画面の四隅にカーソルを移動した際に意図しない動作が発生する場合があります。

**システム設定 → デスクトップとDock → ホットコーナー**

すべて「—」（無効）に設定しておくと誤操作を防げます。

### バッテリー残量をパーセント表示

**システム設定 → コントロールセンター → バッテリー → 割合（%）を表示**

### Mission Control のショートカット確認

Windowsの `Alt + Tab` に相当する操作は `⌘ + Tab`（アプリ切り替え）ですが、**同一アプリ内のウィンドウ切り替え**は `` ⌘ + ` `` です。これはWindowsユーザが見落としがちなショートカットです。

### ブラウザ・画面の拡大・縮小

Windowsでは `Ctrl + +` / `Ctrl + -` / `Ctrl + 0` で行っていたブラウザのズーム操作は、Macでは以下のようになります。

| 操作 | Windows | Mac |
|---|---|---|
| 拡大 | `Ctrl + +` / `Ctrl + マウスホイール↑` | `⌘ + +` / `⌘ + マウスホイール↑` |
| 縮小 | `Ctrl + -` / `Ctrl + マウスホイール↓` | `⌘ + -` / `⌘ + マウスホイール↓` |
| 100%にリセット | `Ctrl + 0` | `⌘ + 0` |

トラックパッドの場合は、**ピンチイン / ピンチアウト**（2本指でつまむ / 広げる）でも拡大・縮小が可能です。

#### 画面全体を拡大する（アクセシビリティズーム）

ブラウザだけでなく**画面全体**を拡大したい場合は、アクセシビリティのズーム機能を使います。

**システム設定 → アクセシビリティ → ズーム機能**

- 「キーボードショートカットを使ってズーム」を **オン**

有効にすると以下のショートカットが使えます。

| ショートカット | 動作 |
|---|---|
| `⌥ + ⌘ + =` | 画面を拡大 |
| `⌥ + ⌘ + -` | 画面を縮小 |
| `⌥ + ⌘ + 8` | ズームのオン/オフ切り替え |

- 「スクロールジェスチャと修飾キーを使ってズーム」を **オン** にすると、`⌃`（Control）キーを押しながらマウスホイールやトラックパッドのスクロールで画面全体をズームできます

<Note>

プレゼンテーションや画面共有で特定の箇所を拡大して見せたい場合にも、アクセシビリティズームは非常に便利です。

</Note>

### よく使うショートカットの対応表

| Windows | Mac | 操作 |
|---|---|---|
| `Ctrl + C / V / X` | `⌘ + C / V / X` | コピー / ペースト / カット |
| `Ctrl + Z` | `⌘ + Z` | 元に戻す |
| `Ctrl + A` | `⌘ + A` | 全選択 |
| `Ctrl + F` | `⌘ + F` | 検索 |
| `Alt + Tab` | `⌘ + Tab` | アプリ切り替え |
| `Alt + F4` | `⌘ + Q` | アプリ終了 |
| `Ctrl + Shift + Esc` | `⌘ + ⌥ + Esc` | 強制終了 |
| `Win + E` | `⌘ + ⌥ + Space` | Finder を開く |
| `Win + D` | `F11` または `fn + F11` | デスクトップ表示 |
| `Win + L` | `⌃ + ⌘ + Q` | 画面ロック |
| `Print Screen` | `⌘ + ⇧ + 3` | スクリーンショット（全画面） |
| `Alt + Print Screen` | `⌘ + ⇧ + 4` | スクリーンショット（範囲選択） |
| `Home / End` | `⌘ + ← / →` | 行頭 / 行末 |
| `Ctrl + Home / End` | `⌘ + ↑ / ↓` | 文書の先頭 / 末尾 |
| `Delete（→方向）` | `fn + Delete` | 右側の文字を削除 |

## まとめ

WindowsからMacへの移行は最初こそ戸惑いますが、適切な設定とアプリの導入で快適に使えるようになります。

ポイントをまとめると：

1. **システム設定** — トラックパッド、Dock、キーリピートを調整
2. **Finder** — パスバー表示、拡張子表示、隠しファイル表示を有効化
3. **Homebrew** — 最初にインストールし、以降はすべて `brew` で管理
4. **Rectangle / Raycast / Maccy** — Windowsの便利機能を補完する三種の神器
5. **Karabiner-Elements** — キーボードカスタマイズの定番
6. **ショートカット** — Command キーと Control キーの違いに慣れる

最初の1週間を乗り越えれば、Macの快適さを実感できるはずです。
